### 12.3 Database Requirements for Feature Tests

**Issue:** Feature tests require a local MySQL database but the current environment only has access to the remote production database.

**Configuration Required (`phpunit.xml`):**
```xml
<env name="DB_CONNECTION" value="mysql"/>
<env name="DB_DATABASE" value="studai_career_testing"/>
<env name="DB_HOST" value="127.0.0.1"/>
<env name="DB_PORT" value="3306"/>
<env name="DB_USERNAME" value="root"/>
<env name="DB_PASSWORD" value=""/>
```

### 12.4 Test Setup Instructions

**Option 1: Local MySQL (Recommended)**
```powershell
# 1. Install MySQL locally or use XAMPP/Laragon
# 2. Create test database:
mysql -u root -p -e "CREATE DATABASE studai_career_testing;"

# 3. Run migrations on test database:
php artisan migrate --env=testing

# 4. Run all tests:
php artisan test
```

**Option 2: Enable SQLite (Requires PHP extension)**
```powershell
# 1. Enable SQLite in php.ini:
# Uncomment: extension=pdo_sqlite
# Uncomment: extension=sqlite3

# 2. Update phpunit.xml:
<env name="DB_CONNECTION" value="sqlite"/>
<env name="DB_DATABASE" value=":memory:"/>

# 3. Run tests:
php artisan test
```

**Option 3: Docker MySQL**
```powershell
# 1. Run MySQL container:
docker run --name studai-mysql-test -e MYSQL_ROOT_PASSWORD=secret -e MYSQL_DATABASE=studai_career_testing -p 3307:3306 -d mysql:8

# 2. Update phpunit.xml with port 3307
# 3. Run tests:
php artisan test
```

### 12.6 PHPUnit Configuration Notes

⚠️ **Deprecation Warning:** Tests use doc-comment annotations (`/** @test */`) which are deprecated in PHPUnit 12. Consider migrating to PHP 8 attributes:

```php
// Current (deprecated in PHPUnit 12):
/** @test */
public function it_can_check_if_agent_can_apply_today()

// Recommended (PHP 8 attributes):
#[Test]
public function it_can_check_if_agent_can_apply_today()
```
